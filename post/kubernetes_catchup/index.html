Kubernetes (K8s)

  • https://kubernetes.io/ja/
  • https://kubernetes.io/ja/docs/setup/

メリット

  • サービスディスカバリー

    サービスディスカバリーによって、サーバーにあるアプリケーションのポート番号やIPアドレスなどを知ることができます。 また、どのようなアプリケーションが動いているのかを知ることもできます。

  • 自己回復機能

    自己回復機能により、コンテナがダウンしてしまったり、間違えて削除してしまったりした場合も自動で回復させることができます。 つまり、Kubernetesは障害に強いといえます。

  • オンプレミスとクラウドで利用できる

    Kubernetesは、オンプレミスとクラウドで運用できる点もメリットです。 自社サーバーでもクラウドでも使えるので、利用する場所を選びません。

  • ストレージの管理

    Kubernetesでは、ストレージの管理ができます。 データはコンテナに記録され、有効期限が定められた状態でPod内に保存されます。

Kubernetes概要

  • 🐳 コンテナ
    • アプリケーションを独立して実行する入れ物
    • 手元のマシン内で複数のアプリケーションを依存関係に囚われず使用できる優れもの。
    • -> Docker
  • ⚓️ コンテナオーケストレーション 
    コンテナを用いてプロダクション運用する場合、
マシンが１つだと、そのマシンが何らかなの故障不具合が起きたら終わり...orz
だから、複数台準備して、マシン毎にコンテナを用いる!
しかし、数が増えたり、マシン要求が複雑化（e.g. 「メモリがいいのがいい〜」「ストレージがいいのがいい〜」...etc.）したら管理がめちゃ大変...
そこで、`Container Orchestration`
  • ⚓️ コンテナオーケストレーションの機能
    1. デプロイメント: 異なるバージョンのコンテナを、どのように入れ替えるか
    2. スケジューリング: どのマシン上で動かすか（地理的分散、マシン要件…）
    3. オートスケーリング：コンテナ数、マシン数の増減
    4. ネットワーク：コンテナ間のロードバランス、通信サービスのディスカバリ
    5. リソースマネジメント：使用可能リソースを環境ごとにデフォルトやリミットを定義
    6. セキュリティ：ネットワークポリシー、リソースの権限定義

kubectl

  • kubectlとは
    • Command line tool
    • yaml や Command を APIリクエストに変換して api-server にHTTPリクエストを送る役割 
      # Pod作成（理想状態）
$ kubectl create --filename sample_pod.yaml
      • -> POST/api/v1/namespaces/default/pods api-server にリクエストされる
    • kubectlインストール
      • K8sのクラスタバージョン確認 (Docker Desktop > Dashboard > ⚙️> kubernetes)
      $ kubectl version
Client Version: v1.30.5-dispatcher
Kustomize Version: v5.0.4-0.20230601165947-6ce0bf390ce3
Server Version: v1.30.5-gke.1014003

      # 現在のクラスタとユーザかを確認
$ kubectl config current-context
docker-desktop
    • kubectlの構文 
      $ kubectl [command] [TYPE] [NAME] [flags]
      • [command] 実行したい操作（e.g. get, create, patch, delete …etc.）
      • [TYPE] リソースタイプ（e.g. pod, node, service, deployment …etc.）
      • [NAME] リソース名
      • [flags] オプションフラグ（e.g. --kubecofig …etc.）
      # 
$ kubectl -n production-data-transform-jp-dev get pod
NAME                                   READY   STATUS             RESTARTS          AGE
production-dbt-docs-5d8b98b74f-ck5zq   0/1     Ready              38 (52s ago)      172m
production-dbt-docs-5d8b98b74f-htgq4   0/1     CrashLoopBackOff   191 (4m39s ago)   16h

      # リソース一覧を表示
$ kubectl api-resources
NAME SHORTNAMES APIVERSION NAMESPACED KIND
pods po         v1         true       Pod
{中略}
      • NAME* リソース名
      • SHORTNAMES* リソース名短縮
      • APIVERSION APIバージョン
      • NAMESPACED 名前空間に属するか
      • KIND* リソース種類
      • * は [TYPE]に付加できる

Kubernetsデバッグ

  • [nord]をデバッグ 
    $ kubectl get nord
  • [pod]をデバッグ 
    # 「Running」状態にないpodを探す
$ kubectl get pod

    # 「describe」でデバック「Events」配下で状況確認できる
$ kubectl describe pod -n your_pod_name
{中略}
Events: ~

# 「-o yaml」でデバック「Status:」の中に状況確認できる
$ kubectl get pod your_pod_name -o yaml
{中略}
Status: ~

    # 失敗が分かれば修正して「apply」
$ kubectl apply -f pod-your_pod_name-image.yaml

# 再度状態確認
$ kubectl get pod
  • [Service]をデバッグ 
    $ kubectl describe endpoints your_pod_name
